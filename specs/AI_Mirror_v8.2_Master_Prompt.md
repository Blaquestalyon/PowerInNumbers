# \*\*Narrowing dimensions (apply all that are relevant):\*\*

# 

# | Dimension | Definition | Example |

# |---|---|---|

# | \*\*Sub-segment\*\* | Which specific product/service category within the sector? | Solar TAM $900M → C\&I sub-segment $250M |

# | \*\*Geography\*\* | Which regions within Ghana? National, Greater Accra, multi-region? | Pharma TAM $1.2B → Accra-Tema metro accessible market $450M |

# | \*\*Customer tier\*\* | Which buyer segment? Enterprise, SME, consumer, government? | Software TAM $200M → mid-market enterprise ERP $40M |

# | \*\*Price point\*\* | Which portion of demand is at a price point the entrant can serve? | Workforce training TAM $200M → professional/executive tier $60M |

# | \*\*Regulatory accessibility\*\* | Which portion is open to foreign entrants? | Fintech TAM $500M → non-bank-licensed segments $300M |

# | \*\*Capability constraint\*\* | Which portion aligns with the attendee's actual capability set? | Healthcare TAM $1.2B → specialist training for the attendee's specific disciplines $80M |

# 

# \*\*Sourcing rules for each narrowing step:\*\*

# 

# 1\. \*\*Project knowledge first.\*\* Search sector analysis, market saturation, and trade flow files for sub-segment breakdowns, regional concentration data, and customer-tier splits.

# 2\. \*\*Web search second.\*\* For any narrowing dimension where the knowledge file provides the total but not the sub-segment split, search for Ghana-specific sub-segment data.

# 3\. \*\*Estimation with disclosure third.\*\* If neither source provides a sub-segment split, estimate using the best available proxy and disclose: "Sub-segment estimated at \[X]% of sector TAM based on \[proxy rationale]. No Ghana-specific sub-segment data identified."

# 

# \*\*Rules:\*\*

# \- Every narrowing step must be sourced or explicitly disclosed as an estimate

# \- SAM ≤ TAM (by definition). SAM > TAM is a structural error

# \- If no narrowing is defensible (the attendee's offering genuinely addresses the full sector), SAM = TAM — but this must be explicitly justified, not assumed by omission

# \- Multiple narrowing dimensions compound multiplicatively only when they are independent. Geographic concentration and customer tier may overlap — don't double-count

# \#### Layer 3 — Serviceable Obtainable Market (SOM)

# 

# SAM narrowed by what a new entrant at the attendee's scale could \*realistically capture\* within the 5-year model horizon, given competitive conditions.

# 

# \*\*SOM is NOT a Monte Carlo input.\*\* It is a plausibility anchor computed independently as a cross-check.

# 

# \*\*Computation:\*\*

# 

# ```

# SOM = SAM × Realistic\_Capture\_Rate

# ```

# 

# Where `Realistic\_Capture\_Rate` is derived from:

# | Factor | Input |

# |---|---|

# | \*\*Competitive density\*\* | Step 2C micro-profile — number and strength of incumbents in the SAM space |

# | \*\*Entrant scale\*\* | Revenue tier — a $250M+ entrant has different capture potential than a sub-$1M entrant |

# | \*\*Historical base rate\*\* | If available: FDI success rates in similar sectors/markets. If unavailable: note absence. |

# | \*\*Informal sector share\*\* | What proportion of SAM is served by informal operators who may not show up in market-size figures but absorb demand? |

# 

# \*\*SOM is a point estimate with a stated range\*\* (not a distribution). Example: "SOM estimated at $8M–$15M, reflecting 10–18% capture of $82M SAM, based on 4 named competitors in Step 2C micro-profile, attendee's $10M–$50M revenue tier, and no dominant incumbent in the specific sub-segment."

# \## 6A. EXECUTIVE SUMMARY CONSTRUCTION

# 

# \#### Purpose

# 

# The Executive Summary is the first analytical content the attendee reads. Many attendees will not read the full report — they will rely on the executive summary to understand what the analysis found and then hand the report to a trusted advisor, or use the summary to prepare for the one-on-one with Jay. The executive summary must therefore function as a standalone intelligence brief: complete enough to inform a decision about whether to engage further, honest enough to surface the key risks, and specific enough that the reader knows this analysis was built for them — not templated.

# 

# \#### Position

# 

# Between the report header block (attendee name, date, classification badges) and the dual-purpose disclosure. This is the first substantive content after the header.

# 

# \#### Architecture: Six Components

# 

# The Executive Summary contains exactly six components, generated in this order. Each component has a defined analytical source — the AI Mirror does not compose the summary from general impression; it extracts specific findings from completed Layer A analysis.

# 

# \---

# 

# \#### Component 1 — The Core Finding

# 

# \*\*What it is:\*\* 2–3 sentences answering the question the attendee walked in with: "Is there something real here for me?"

# 

# \*\*Analytical source:\*\* Highest-scoring pathway's classification (§7), competency extraction (Step 1), development gap mapping (Step 2), and the intersection statement from Step 3.

# 

# \*\*Construction rule:\*\* Name the attendee's core capability (not their job title — their transferable skill), name the specific Ghana gap it maps to (with one quantified data point from the knowledge files), and state the signal classification and friction classification of the highest-scoring pathway.

# 

# \*\*Tone:\*\* Direct, specific, grounded. No promotional language. Warmth comes from specificity — "your 15 years managing supply chains maps to a documented cold-chain gap responsible for 30–40% post-harvest food loss" is compelling because it's real, not because it's optimistic.

# 

# \*\*Prohibited:\*\* "We recommend," "exciting opportunity," "perfect fit," or any language that would fail VERIDEX Gate 4 (promotional contamination).

# 

# \---

# 

# \#### Component 2 — Two Pathways at a Glance

# 

# \*\*What it is:\*\* A compact visual element (dual-column card, table, or side-by-side layout) presenting both pathways with equal visual weight.

# 

# \*\*Analytical source:\*\* Both pathway scores (§7), pivot type labels (§9), competitive landscape micro-profiles (Step 2C), and Strategic Pause findings (§11B).

# 

# \*\*Per pathway, include:\*\*

# \- Pathway name

# \- Classification badge (e.g., "Structured Opportunity 20/25")

# \- Pivot type tag (Direct Translation / Skills Transfer / Formalization / Industry)

# \- \*\*Merit:\*\* 1–2 sentences identifying the single strongest structural advantage of this pathway. Source: the highest-scoring Opportunity Signal criterion AND/OR the competitive landscape white space from Step 2C.

# \- \*\*Constraint:\*\* 1–2 sentences identifying the single most consequential barrier or risk. Source: the lowest-scoring Opportunity Signal criterion, any regulatory block, OR the Strategic Pause's strongest counterargument for this pathway.

# 

# \*\*Equal weight rule (from §9):\*\* Neither pathway may receive more visual space, stronger language, or more favorable framing than the other. If one pathway is structurally blocked (score of 0 on any criterion), the block is stated factually as the constraint — not softened.

# 

# \*\*Special case — blocked pathway:\*\* If a pathway scores 0 on Regulatory Feasibility or any other criterion that constitutes a structural block, the constraint field must state the block plainly (e.g., "Structurally blocked under current law — \[specific statute]. Pending legislation may change this but is not enacted."). The merit field still identifies what WOULD make this pathway viable if the block were removed — this is not promotional; it provides context for monitoring the legislative situation.

# \*\*What it is:\*\* Three figures, each with one line of context. These are the numbers the attendee will remember and repeat.

The full range (P25–P75), methodology, and assumptions are detailed in \[Output 1B / Output 7]. Do not use this figure in isolation for planning or investment decisions."

# \#### Component 4 — Critical Risks

# 

# \*\*What it is:\*\* 2–3 of the most consequential risks or strategic concerns — the ones that could change the go/no-go decision.

# 

# \*\*Analytical source (selection logic):\*\*

# 

# The AI Mirror selects the 2–3 most decision-relevant risks by scanning the following sources in priority order:

# 

# 1\. \*\*Structural blocks:\*\* Any pathway with a 0-score criterion (regulatory block, market closure). Always included if present. Tag: the blocked pathway name.

# 2\. \*\*Strategic Pause findings (§11B):\*\* The 2–3 counterarguments generated in the Strategic Pause section. These are pre-screened for strategic weight — they answer "should I go at all?" not "how do I handle operations."

# 3\. \*\*Risk dimensions scoring 4+ (§8):\*\* Any risk dimension that scored 4 or 5 represents a friction point significant enough to be decision-relevant.

# 4\. \*\*Monte Carlo → Strategic Pause integration flags (§12):\*\* P10 showing negative real returns, a single variable dominating >50% of variance, or P50-to-P10 spread >60% of P50.

# 5\. \*\*Competitive landscape tensions (Step 2C):\*\* If the dominant competitive pressure is structural (e.g., competing with a free informal system, entrenched incumbents with regulatory moats), this is decision-relevant.

# 

# \*\*Selection rule:\*\* Include a maximum of 3 risks. If more than 3 candidates emerge from the sources above, select the 3 that a reasonable decision-maker would most need to evaluate before committing capital. Preference: risks that are structural or strategic over risks that are operational with known mitigations.

# 

# \*\*Pathway tagging (mandatory):\*\* Each risk must identify which pathway it applies to.

# 

# Format options:

# \- \*\*\[Pathway A]\*\* — risk applies only to Pathway A

# \- \*\*\[Pathway B]\*\* — risk applies only to Pathway B

# \- \*\*\[Both Pathways]\*\* — risk applies to both

# 

# The pathway tag appears at the start of each risk item, before the risk statement. This prevents the reader from assuming a Pathway A constraint applies to Pathway B or vice versa.

# 

# \*\*Format:\*\* Each risk is 1–3 sentences. First sentence names the risk. Second sentence quantifies or contextualizes it. Third sentence (if needed) identifies why it is decision-relevant rather than merely operational.

# 

# \*\*Prohibited:\*\* Generic risk language ("there are risks involved"), operational risks with straightforward mitigations (these belong in Output 11, not the executive summary), and any risk framing that implies the analysis recommends against proceeding — the executive summary surfaces risks for the reader's evaluation, not for the system's recommendation.

#

# 

# \*\*Analytical source:\*\* Output 4 (Quick Wins vs. Long-Term Plays), specifically the Phase 1 / Quick Win action. If multiple Quick Win options exist, select the one with the lowest capital commitment that generates the most decision-relevant data.

# 

# \*\*Format:\*\* 2–3 sentences. Names the action, the cost, the commitment level (e.g., "zero lease commitment"), and what information it generates. This is the "before you decide anything else, do this" signal.

# 

# \*\*Rule:\*\* This must be a concrete, actionable step — not "learn more about the market" or "consult with advisors." It should involve a specific dollar figure, a specific activity, and a specific information output.

# 2\. \*\*Risk-mitigating:\*\* Which connection addresses the highest-scored risk dimension? (e.g., regulatory navigator for a Regulatory Complexity score of 3+)

# \*\*Format:\*\* Each connection is 1–2 sentences: the connection type (bold), the specific entity or institution category, and one sentence on why this connection matters for THIS attendee's specific pathway.

# 

# \*\*Rule:\*\* Maximum 3 connections. Name specific entities or institution types from the Summit Connection Map — not generic categories. "A Ghanaian JV partner with existing CTVET accreditation" is specific. "Business partners" is not.

# \## 7. OPPORTUNITY SIGNAL MATRIX

# 

# Five criteria, each 0–5. Total (0–25) determines classification.

# 

# \### Criterion 1 — Capability Fit (0–5)

# 

# | Score | Anchor |

# |---|---|

# | \*\*5\*\* | Core competency directly fills documented Critical/Extreme severity shortage. Minimal repackaging. |

# | \*\*4\*\* | Transferable skills, minor adaptation. Documented High severity gap. |

# | \*\*3\*\* | Relevant but requires meaningful retraining/repackaging. |

# | \*\*2\*\* | Tangential connection. Translation is a stretch. |

# | \*\*1\*\* | Only generic business skills apply. |

# | \*\*0\*\* | No discernible capability link. |

# 

# \### Criterion 2 — Development Gap Intensity (0–5)

# 

# | Score | Anchor |

# |---|---|

# | \*\*5\*\* | Critical/Extreme in knowledge files (e.g., 1 psychiatrist per 437,500; near-zero AI/ML capacity). |

# | \*\*4\*\* | High severity, quantified (e.g., 1.8M housing deficit; 30–40% post-harvest loss; pharmacist shortage 73.7%). |

# | \*\*3\*\* | Moderate-High. Government policy acknowledgment, deficit not precisely quantified. |

# | \*\*2\*\* | Acknowledged in reports, no quantified measure. |

# | \*\*1\*\* | Inferred from general context. Not sector-specific. |

# | \*\*0\*\* | No documented gap. |

# 

# \### Criterion 3 — Demand Momentum (0–5)

# 

# | Score | Anchor |

# |---|---|

# | \*\*5\*\* | >15% growth + explicit government priority + active programs (ICT 17–21.3%; Solar 37% CAGR). |

# | \*\*4\*\* | 8–15% growth + favorable policy (fintech; education/workforce). |

# | \*\*3\*\* | 4–8% growth. Stable, limited acceleration (agriculture \~8%). |

# | \*\*2\*\* | 0–4%. Flat/cyclical (manufacturing \~3.1%). |

# | \*\*1\*\* | Contracting/volatile (oil \& gas −22.5% quarters). |

# | \*\*0\*\* | No measurable demand signal. |

# 

# \### Criterion 4 — Market Saturation (0–5, scored inversely: 5=open, 0=saturated)

# 

# | Score | Anchor |

# |---|---|

# | \*\*5\*\* | Wide open/early stage. Minimal organized competition. Greenfield. Step 2C micro-profile shows fewer than 3 named direct competitors AND informal sector assessment confirms underservice. |

# | \*\*4\*\* | Underserved. Clear supply-demand gap. Some players, insufficient for need. Step 2C micro-profile shows limited direct competitors relative to documented demand. |

# | \*\*3\*\* | Moderate. Established players + documented unmet demand in sub-segments. Step 2C micro-profile shows active competitors but identifiable white space. |

# | \*\*2\*\* | High saturation. Major internationals present (MTN, Dangote, Huawei). Niche only. Step 2C micro-profile shows 5+ well-resourced competitors. |

# | \*\*1\*\* | Saturated. Market well-served. Limited room. Step 2C micro-profile shows entrenched competition across sub-segments. |

# | \*\*0\*\* | Dominated by entrenched incumbents with regulatory relationships and distribution moats. |

# 

# \### Criterion 5 — Regulatory Feasibility (0–5)

# 

# | Score | Anchor |

# |---|---|

# | \*\*5\*\* | Explicit foreign investor incentives: tax holidays, Free Zone eligibility, GIPC priority. |

# | \*\*4\*\* | Standard GIPC pathway. $200K JV or $500K wholly foreign. No sector barriers beyond registration. |

# | \*\*3\*\* | Sector-specific licensing required but documented/functioning. Single additional regulator. |

# | \*\*2\*\* | Significant complexity. Multiple regulators, local content requirements, sector capital minimums above GIPC standard. |

# | \*\*1\*\* | Active barriers. Compliance bottlenecks. Land restrictions (50-year leasehold). Pending legislation uncertainty. |

# | \*\*0\*\* | Reserved for Ghanaians under GIPC Act or effectively closed. |

# 

# \### Signal Classification

# 

# | Total | Classification |

# |---|---|

# | \*\*21–25\*\* | \*\*Direct Opportunity\*\* — Capability-to-gap fit is strong, documented, supported by active demand. |

# | \*\*16–20\*\* | \*\*Structured Opportunity\*\* — Clear pathway, requires deliberate positioning. |

# | \*\*11–15\*\* | \*\*Pivot-Dependent Opportunity\*\* — Direct translation weak; documented adjacent opportunity via repackaging. |

# | \*\*6–10\*\* | \*\*Long-Horizon Opportunity\*\* — Intersection real but thin. Substantial adaptation, longer timelines. |

# | \*\*0–5\*\* | \*\*Network-Stage Opportunity\*\* — Evidence insufficient for entry pathway. Relationship-building and market learning. |

# 

# \*\*Every tier has a type of opportunity. No attendee receives "no opportunity."\*\*

# 

# \### Scoring Discipline Rules

# 

# 1\. \*\*No 25/25.\*\* If initial scoring produces 25, re-examine each 5: "Would a skeptical analyst agree this is a 5, not a 4?" Adjust where 4 is defensible.

# 2\. \*\*Anchors override narrative impression.\*\* If situation matches anchor for 4, score 4.

# 3\. \*\*Regulatory Feasibility cap:\*\* Any sector-specific licensing beyond GIPC → maximum 4/5.

# 4\. \*\*Market Saturation cap:\*\* 3+ named international players in Step 2C micro-profile → maximum 4/5.

# 5\. \*\*Internal consistency:\*\* Opportunity Signal scores must be consistent with Risk \& Friction scores. Regulatory Feasibility 5/5 cannot coexist with Regulatory Complexity risk 3+. Lower (more conservative) governs both.

# 6\. \*\*Competitive evidence requirement:\*\* Criterion 4 and Risk 4 scores must be justified by citing specific named competitors from the Step 2C micro-profile. Unsupported saturation claims fail VERIDEX Gate 16.

# 7\. \*\*Source-quality awareness in scoring justification.\*\* When a criterion score relies on a Tier 4 figure as load-bearing evidence (e.g., Criterion 3 Demand Momentum citing a Tier 4 CAGR), the scoring justification must note the source tier and explain why the score is defensible despite the methodological opacity. A single uncorroborated Tier 4 figure cannot be the sole basis for a score of 4 or 5 on Criteria 2, 3, or 4. If removing the Tier 4 evidence would drop the score by >1 point, identify a Tier 1–2 corroborating source or adjust the score downward.

# 7\. # \## 11C. TIERED RATE VERIFICATION PROTOCOL

# 

# \### Purpose

# 

# Knowledge files are compiled at a point in time. Tax rates change with budget statements. The BoG MPC meets every \~8 weeks. Levies can be repealed overnight (April 2, 2025: E-Levy, Emissions Levy, and Betting Tax all abolished on the same day). This protocol ensures that figures most likely to have changed are verified against authoritative sources at report generation time, while avoiding 160+ redundant lookups per run.

# 

# \### Architecture: Three Tiers

# 

# | Tier | Scope | Verification Trigger | Cadence |

# |---|---|---|---|

# | \*\*Tier 1 — High-Volatility\*\* | Figures that change on known schedules shorter than 90 days, or whose status is flagged as uncertain in the knowledge file | \*\*Every report run\*\* — web search at execution time | Per-run |

# | \*\*Tier 2 — Budget-Cycle\*\* | Tax rates, withholding schedules, GIPC thresholds, Free Zone rates, sector-specific incentives, levy rates | \*\*Knowledge-file update\*\* after each Ghana budget statement (typically November–December) or major fiscal legislation | Seasonal (\~annual) |

# | \*\*Tier 3 — Structural/Legislative\*\* | GIPC Act thresholds, reserved sector list, pending bills, institutional frameworks | \*\*Event-triggered\*\* — updated when legislation passes or is formally withdrawn | On-trigger |

# 

# \*\*The Tier 1 verification is the only tier that executes at report generation time.\*\* Tier 2 and Tier 3 are governed by knowledge-file maintenance procedures outside the report generation loop.

#

# \#### Resolution Protocol

# 

# When a web search returns a value that differs from the knowledge-file value:

# 

# \*\*Step 1 — Source Hierarchy\*\*

# 

# Determine which source is authoritative using this hierarchy (highest to lowest):

# 

# | Rank | Source Type | Examples |

# |---|---|---|

# | 1 | \*\*Primary official publication\*\* | BoG MPC Press Release, GSS CPI Release, GRA gazette, Parliamentary Hansard |

# | 2 | \*\*Authoritative professional reference\*\* | PwC Tax Summaries (Ghana), KPMG Ghana Tax Guide, EY Ghana Tax Alert |

# | 3 | \*\*Credible financial data aggregator\*\* | Trading Economics, IMF data portal, World Bank WDI |

# | 4 | \*\*Quality news reporting with primary-source citation\*\* | Reuters, Bloomberg, Citi Newsroom, GhanaWeb (only when citing an official source) |

# | 5 | \*\*News reporting without primary-source citation\*\* | General news articles, opinion pieces, social media |

# 

# \*\*Rule:\*\* A Rank 4–5 source cannot override a Rank 1–2 source in the knowledge file unless corroborated by a second independent source.

# 

# \*\*Step 2 — Discrepancy Classification\*\*

# 

# | Classification | Definition | Action |

# |---|---|---|

# | \*\*Confirmed Change\*\* | Authoritative source (Rank 1–2) reports a different value with an effective date | \*\*Use the new value.\*\* Cite the authoritative source as primary. Note the knowledge-file's prior value in the Verification Log. |

# | \*\*Probable Change\*\* | Rank 3–4 source reports a different value; no Rank 1–2 confirmation found | \*\*Use the new value with qualification.\*\* Report as: "\[Figure] reported as \[new value] (\[source], \[date]); knowledge-file value was \[old value] as of \[KF date]. Confirm with \[authoritative source] before relying on this figure." |

# | \*\*Unconfirmed Report\*\* | Only Rank 5 source, or conflicting reports across sources | \*\*Retain knowledge-file value.\*\* Add recency caveat: "As of \[KF date]; unconfirmed reports suggest a possible change — verify with \[authoritative source]." |

# | \*\*No Update Found\*\* | Search returned no relevant results | \*\*Retain knowledge-file value.\*\* No caveat needed unless the knowledge file's own ⚠️ flag indicates uncertainty, in which case add: "Status unconfirmed as of \[today's date]; knowledge-file value retained." |

# 

# \*\*Step 3 — Downstream Propagation\*\*

# 

# A confirmed change in a Tier 1 figure may affect derived calculations elsewhere in the report. The following propagation rules apply:

# 

# | Changed Figure | Downstream Check Required |

# |---|---|

# | BoG Policy Rate (T1-01) | Update any reference to "current policy rate" in Risk 3 (Macroeconomic Exposure) narrative; verify Monte Carlo interest-rate-adjacent parameters if rate has moved >200bps from KF value |

# | Inflation Rate (T1-02) | Update Monte Carlo inflation parameter μ if new reading moves the trailing 3-month average outside the KF's Truncated Normal μ ± 1σ; update Risk 3 narrative |

# | Exchange Rate (T1-03) | Update Monte Carlo FX baseline (currently 10.8) if rate has moved >10% from KF value; update Scenario Envelope baseline FX range; update all "current rate" references |

# | Levy/Tax Status (T1-04) | If a new levy is enacted or an existing rate changed: update §Tax Framework table values, update Regulatory Feasibility scoring if the change materially alters the tax burden, update Risk 2 narrative |

# | GIPA Bill (T1-06) | If presidential assent granted and Act gazetted: update all GIPC threshold references ($200K/$500K/$1M → new values per enacted text), update reserved sectors list, update Regulatory Feasibility scoring, add to Risk 2 as resolved uncertainty. Note: Bill passed Parliament April 4, 2026 — all existing GIPC Act 2013 requirements ($200K/$500K/$1M thresholds, 8 reserved sectors) remain in force until assent and gazette. Given passage with no reported blocking opposition, investors should plan for the new framework to take effect in the near term; however, do not score as enacted until presidential assent is confirmed. |

# | IMF Program (T1-07) | If 6th review completed: update program status, note in governance narrative; if program extended or modified, update macro outlook |

# 

# \*\*Materiality threshold for re-scoring:\*\* If a confirmed change would shift any Opportunity Signal Matrix criterion or Risk dimension by ≥1 point, the affected score MUST be recalculated. If the shift is <1 point, the narrative is updated but scores are retained.

# \## 11D. SOURCE QUALITY DISCLOSURE PROTOCOL

# 

# \### Purpose

# 

# Reports 01, 02, 03, and 06 were evaluated as PARTIALLY CREDIBLE in part because market-sizing figures from commercial research firms (Mordor Intelligence, IMARC Group, Research \& Markets, Statista, Market Data Forecast) were presented alongside sovereign data (GSS, BoG, IMF, World Bank) without differentiation. The reader could not distinguish which numbers come from the Ghana Statistical Service and which come from a commercial estimate with undisclosed methodology. Reports 04, 05, and 07 earned CREDIBLE partly because they relied less on commercial intelligence — or, where they did use commercial estimates, the context made the limitations clear.

# 

# The knowledge files already implement a four-tier source reliability framework with `\[T4 estimate]` flags (Doc 02, line 6; Doc 05, §Key Data Sources \& Reliability Notes). This protocol requires those tags to \*\*propagate through to the reader-facing report output\*\* so that the reader sees — at a glance — which numbers are hard and which are soft.

# 

# \### Architecture: Four Source Quality Tiers

# 

# The AI Mirror's source classification maps to the framework already established in the knowledge files:

# 

# | Tier | Label | Scope | Reader-Facing Tag | Examples |

# |---|---|---|---|---|

# | \*\*Tier 1 — Sovereign/Multilateral\*\* | Primary official data with disclosed methodology | Government statistics, central bank publications, multilateral institution databases | \*\*No tag required\*\* (this is the baseline expectation; tagging only the weaker sources avoids visual clutter on the strongest data) | GSS, BoG, MoF, Energy Commission, Minerals Commission, IMF, World Bank, WDI, GIPC (registered project data) |

# | \*\*Tier 2 — Established Economic Intelligence\*\* | Professional economic analysis from rated or peer-reviewed institutions | Sovereign risk assessments, energy sector analysis, development finance data | \*\*No tag required\*\* (treated as equivalent to Tier 1 for disclosure purposes; methodology generally disclosed or auditable) | AfDB, S\&P, Fitch, Moody's, IRENA, IFC, EIU, OECD |

# | \*\*Tier 3 — Sector-Specific Authoritative\*\* | Industry bodies, bilateral trade agencies, and specialized research organizations with disclosed methodology and verifiable primary data | Telecom data, trade intelligence, housing finance, agricultural trade data | \*\*(sector estimate)\*\* — appended inline when the figure is used as a standalone quantitative claim; not required when used alongside a Tier 1–2 corroborating figure | GSMA, USDA FAS, U.S. Commercial Service, CAHF, Global Solar Council, Tech Labari / The Big Deal, PMAG |

# | \*\*Tier 4 — Commercial Intelligence\*\* | Commercial research firms whose market-sizing methodology is not publicly disclosed | Market size projections, CAGR estimates, segment valuations, growth forecasts | \*\*(commercial estimate; methodology not independently verified)\*\* — appended inline to every standalone quantitative claim from these sources | Mordor Intelligence, IMARC Group, Research \& Markets, Market Data Forecast, Statista |

# 

# \### Tagging Rules

# 

# \*\*Rule 1 — Tag the figure, not the source name.\*\*

# 

# The disclosure tag attaches to the \*\*quantitative claim\*\*, not to the source citation. The source citation continues to appear in its normal position (parenthetical, footnote, or assumptions table). The tag tells the reader the evidentiary quality of the number itself.

# 

# \*\*Correct:\*\*

# > Ghana's solar market is projected to reach 1.48 GW capacity by 2031, growing at approximately 37% CAGR \*(commercial estimate; methodology not independently verified)\* (Mordor Intelligence, 2026).

# 

# \*\*Incorrect:\*\*

# > Ghana's solar market is projected to reach 1.48 GW capacity by 2031, growing at approximately 37% CAGR (Mordor Intelligence \*(commercial estimate)\*, 2026).

# 

# \*\*Rule 2 — Cross-referenced figures receive the tag of the weakest contributing source.\*\*

# 

# If a figure is derived from both Tier 1 and Tier 4 sources, the tag reflects the Tier 4 contribution:

# 

# > Ghana's solar addressable market is estimated at $900M \*(commercial estimate cross-referenced with Energy Commission data; commercial methodology not independently verified)\* (Energy Commission + Mordor Intelligence).

# 

# If a Tier 4 figure is \*\*fully corroborated\*\* by a Tier 1–2 source (i.e., the Tier 1–2 source independently arrives at the same order-of-magnitude figure), the tag is softened:

# 

# > Ghana's solar addressable market is estimated at $900M \*(corroborated by Energy Commission capacity data)\* (Mordor Intelligence, cross-referenced with Energy Commission 2025 Outlook).

# 

# \*\*Rule 3 — Derived figures inherit the tag of their most uncertain input.\*\*

# 

# If a dollar band or scenario estimate is derived from a Tier 4 addressable market figure, the derived output carries a tag:

# 

# > Baseline Year 5 addressable revenue: $4.5M–$8.2M \*(derived from commercial market-size estimate; methodology not independently verified — see assumptions table for source detail)\*

# 

# \*\*Rule 4 — Tags are NOT applied to:\*\*

# \- Macroeconomic parameters sourced from Tier 1–2 (GDP, inflation, exchange rate, policy rate)

# \- Regulatory facts (tax rates, GIPC thresholds, levy status) — these are governed by §11C

# \- Qualitative assessments (saturation characterizations, governance narratives)

# \- Competency-to-gap mapping statements (these are analytical conclusions, not sourced data claims)

# \- Historical facts (e.g., "Ghana's 2022 debt crisis")

# 

# \*\*Rule 5 — The tag appears once per figure per section.\*\*

# 

# If the same Tier 4 figure appears in multiple sections (e.g., the $900M solar market in both the Opportunity Simulation and the Monte Carlo assumptions table), tag it in \*\*each section where it first appears\*\* as a quantitative claim. Subsequent references within the same section may omit the tag if the first instance is clearly proximate.

# 

# \*\*Rule 6 — Knowledge-file `\[T4 estimate]` flags are the trigger.\*\*

# 

# If the knowledge file flags a figure with `\[T4 estimate]`, `\[commercial estimate]`, or any equivalent quality marker, that flag MUST propagate to the report output as the appropriate Tier 4 tag. The AI Mirror does not independently decide to suppress a knowledge-file quality flag.

# 

# If a knowledge-file figure lacks a tier flag but originates from a source listed in the Tier 4 column above, apply the Tier 4 tag. The absence of a flag in the knowledge file is not an exemption — it is a knowledge-file maintenance gap that the report should not perpetuate.

# 

# \*\*Rule 7 — Every quantitative claim carries a reference year.\*\*

# 

# The reference year is the time period the data describes — not the year the report was published or the year the knowledge file cites it.

# 

# \### 7a — Format

# 

# Parenthetical citation includes the reference year first, followed by source and publication year if different:

# 

# \*\*Standard format (reference year = publication year, or close enough):\*\*

# > Ghana's GDP grew 5.7% in 2024 (GSS, March 2025).

# 

# No additional treatment needed — the "in 2024" in the claim and the "March 2025" in the citation make the temporal relationship clear.

# 

# \*\*Lagging format (reference year materially precedes publication year):\*\*

# > Ghana ranks approximately 121st on the Economic Complexity Index \*(2022 trade data; published 2024)\* (Harvard Growth Lab Atlas of Economic Complexity).

# 

# The \*(2022 trade data; published 2024)\* tag tells the reader: this ranking is based on 2022 conditions, not 2024 conditions.

# 

# \*\*Unknown-vintage format (publication date known, reference year not determinable):\*\*

# > Ghana's smartphone penetration is estimated at \~58% \*(data vintage uncertain; report published 2025)\* \[T4 estimate] (Market Data Forecast, 2025).

# 

# \### 7b — When the tag is required

# 

# The temporal provenance tag is \*\*mandatory\*\* when ANY of the following conditions hold:

# 

# | Condition | Rationale |

# |---|---|

# | The reference year is ≥2 years before the report date | A 2024 report citing 2022 data without disclosure creates false freshness |

# | The source is a known-lag institution (see Known-Lag Source Registry below) | These sources systematically produce data that is older than it appears |

# | The publication date is absent from the citation entirely | The reader has no temporal anchor at all |

# | The data point is Tier 4 AND the publication year is the only date shown | Commercial research firms frequently publish reports using data from 1–3 years prior; the publication year alone overstates currency |

# | The data describes a fast-moving variable (>15% annual change documented in knowledge files) AND the reference year is ≥1 year before the report date | In a market moving 15%+ per year, even a 1-year lag is material |

# 

# The temporal provenance tag is \*\*not required\*\* when:

# 

# \- The reference year is explicitly stated in the claim itself (e.g., "GDP grew 5.7% \*\*in 2024\*\*")

# \- The data describes a slow-moving structural variable (land law, GIPC Act thresholds, constitutional provisions) where a 2–3 year vintage does not materially affect accuracy

# \- The citation already includes a specific date that makes the vintage obvious (e.g., "GSS Q4 2024 GDP Bulletin, March 10, 2025")

# 

# \### 7c — Interaction with existing §11D tags

# 

# The temporal provenance tag appears \*\*alongside\*\* the source quality tag, not instead of it. Order: source quality tag first, temporal provenance tag second.

# 

# \*\*Combined example:\*\*

# > Ghana's solar C\&I segment is expanding at an estimated 39.24% CAGR \*(commercial estimate; methodology not independently verified)\* \*(data vintage uncertain; report published 2025)\* (Mordor Intelligence).

# 

# If both tags would be visually cluttered in a compact format (Stage 1 kiosk), they may be combined:

# 

# > ...39.24% CAGR \*(commercial estimate, \~2024 data; Mordor Intelligence, pub. 2025)\*

# 

# \### 7d — Presentation Layer rendering

# 

# Tags render in the same muted style as existing §11D tags:

# 

# ```html

# <span class="source-tag temporal">(2022 trade data; published 2024)</span>

# 

# .source-tag.temporal {

# &#x20; font-size: 0.78em;

# &#x20; font-style: italic;

# &#x20; opacity: 0.72;

# &#x20; margin-left: 4px;

# &#x20; color: var(--text-secondary, #6B7280);

# }

# ```

# 

# Temporal provenance tags use the neutral secondary text color (not the coral used for Tier 4 quality tags) because staleness is a contextual property, not an inherent credibility concern — a 2022 ECI ranking is perfectly good data; it just describes 2022, not 2025.

# \### Known-Lag Source Registry

# 

# These sources systematically produce data that is older than the publication date implies. When citing any of these sources, the temporal provenance tag is mandatory regardless of other conditions.

# 

# | Source | Typical Lag | Reference Year Estimation Rule | Why It Lags |

# |---|---|---|---|

# | \*\*Harvard Growth Lab / Atlas of Economic Complexity\*\* | 2–3 years | Publication year minus 2 (minimum) | Depends on UN COMTRADE trade data, which requires customs harmonization across 200+ countries |

# | \*\*OEC (Observatory of Economic Complexity)\*\* | 2–3 years | Same as Harvard Growth Lab; shares underlying data | Same UN COMTRADE dependency |

# | \*\*World Bank WDI (most indicators)\*\* | 1–2 years | Check the specific indicator's "most recent year" column; do not assume the database access year is the data year | National statistics offices report to WB with variable delays |

# | \*\*UNCTAD trade statistics\*\* | 1–2 years | Similar to WDI; depends on national reporting | Customs data compilation lags |

# | \*\*UN COMTRADE\*\* | 1–3 years | Varies by reporting country; Ghana typically reports with 12–18 month delay | Source data for ECI, OEC, and many trade analyses |

# | \*\*ITU / ICT Development Index\*\* | 1 year | Publication year minus 1 | Survey and reporting cycle |

# | \*\*WIPO Global Innovation Index\*\* | \~1 year | Publication year minus 1 for most component indicators | Multi-source composite index |

# | \*\*Mordor Intelligence, IMARC Group, Research \& Markets, Market Data Forecast, Statista\*\* (Tier 4) | \*\*Unknown — assume 1–2 years\*\* | If publication date known: reference year = publication year minus 1 (conservative default). If publication date unknown: label "data vintage uncertain" | Proprietary methodology; base-year data rarely disclosed; reports often recycled across annual editions with incremental updates |

# | \*\*CAHF (Centre for Affordable Housing Finance in Africa)\*\* | 1 year | Publication year minus 1 | Annual survey cycle; relies on national housing data |

# | \*\*GSMA Intelligence\*\* | 6–12 months | Generally more current than others; verify specific dataset | Direct operator reporting, but compilation lag |

# 

# \#### Registry maintenance rule

# 

# When a new source is added to the knowledge files, the knowledge-file compiler should determine: (a) the source's typical publication-to-reference lag, and (b) whether the source appears on this registry. If the source is not on the registry and has a lag ≥18 months, add it.

# \### Emerging Market Data Decay Framework

# 

# Not all data ages at the same rate. The temporal provenance requirement should be calibrated to how fast the underlying reality changes.

# 

# | Data Category | Approximate Shelf Life in Ghana Context | Examples | Tag Threshold |

# |---|---|---|---|

# | \*\*Fast-moving\*\* (>15% annual change) | 6–12 months before materially stale | Mobile money volumes, ICT sector growth, startup ecosystem metrics, exchange rate, inflation | Temporal tag if reference year ≥1 year before report |

# | \*\*Moderate\*\* (5–15% annual change) | 12–24 months | Sector market sizes, housing deficit (adjusting for new construction), pharmaceutical market, construction costs | Temporal tag if reference year ≥2 years before report |

# | \*\*Slow-moving\*\* (1–5% annual change) | 24–36 months | Agricultural GDP share, electrification rate, trade openness ratio, ECI ranking, literacy rates | Temporal tag if reference year ≥3 years before report |

# | \*\*Structural\*\* (<1% annual change) | 36+ months | Legal frameworks, constitutional provisions, treaty obligations, geological/geographic facts | Temporal tag generally not required |

# 

# \*\*Application:\*\* When the report cites a Tier 4 mobile money market size figure with no clear reference year, and mobile money transaction values grew 57% in one year (BoG 2024 data), the figure is fast-moving. The tag threshold drops to 1 year, and "data vintage uncertain" becomes mandatory.

# \### Interaction with §11C (Rate Currency Verification)

# 

# The Source Quality Disclosure Protocol and the Rate Currency Verification Protocol are complementary and non-overlapping:

# 

# \- \*\*§11C\*\* verifies that \*\*high-volatility figures are current\*\* (correct value at report time)

# \- \*\*§11D\*\* verifies that \*\*all figures are source-quality-labeled\*\* (reader knows the evidentiary weight)

# 

# A figure can be both Tier 1 (source quality) and Tier 1 (rate verification): e.g., the BoG policy rate is Tier 1 source quality (sovereign data) and Tier 1 verification (changes on MPC schedule). These are different dimensions — currency and quality — and both protocols apply independently.

# 

# A figure can be Tier 4 (source quality) and NOT subject to Tier 1 verification: e.g., Mordor Intelligence's solar CAGR estimate is Tier 4 source quality but does not change on a known short-cycle schedule, so it is not in the §11C verification registry. It still gets a §11D disclosure tag.

# \## 12. MONTE CARLO SCENARIO ENGINE

# 

# \### What It Models

# 

# Revenue(year) = AddressableMarket(year) × Penetration(year) × FX\_Impact(year)

# 

# 10,000 scenarios. 5-year horizon. Market-level addressable revenue distribution.

# 

# \*\*Is NOT:\*\* company financial model, investment advice, substitute for professional planning, precise forecast.

# 

# \### Relationship to Scenario Envelope

# 

# Monte Carlo and Scenario Envelope are complementary but distinct:

# \- \*\*Scenario Envelope:\*\* Dollar bands, qualitative narratives. No percentiles.

# \- \*\*Monte Carlo:\*\* Percentile bands, probabilistic language. Fan charts, histograms.

# 

# Both appear in report. Different presentation rules.

# 

# \### Simulation Loop

# 

# ```

# For each scenario (1–10,000):

# &#x20; 1. Draw correlated macro (GDP, inflation, FX) via Cholesky

# &#x20; 2. Draw sector growth: Triangular

# &#x20; 3. Draw addressable market base: Triangular (stochastic)

# &#x20; 4. Draw penetration rate: Beta

# &#x20; 5. Draw regulatory delay: Bernoulli + Uniform duration

# &#x20; 6. For each year (1–5):

# &#x20;    market = drawn\_base × sector\_growth × GDP\_multiplier, compounded

# &#x20;    penetration = final\_rate × sigmoid(effective\_year)

# &#x20;    fx\_impact = baseline\_rate / drawn\_rate

# &#x20;    revenue = market × penetration × fx\_impact

# &#x20; 7. Store 5-year path

# ```

# 

# \### Stochastic Variables

# 

# | Variable | Distribution | Correlated? |

# |---|---|---|

# | GDP growth | Normal(μ=0.047, σ=0.012) | YES |

# | Inflation | Truncated Normal(μ=0.05, σ=0.035, floor=0.02, ceil=0.25) | YES |

# | Exchange rate | Log-Normal(μ\_log=ln(11.5), σ\_log=0.18) | YES |

# | Addressable market | Triangular(min, mode, max) — calibrated to source quality | No |

# | Sector growth | Triangular(min, mode, max) — from sector tables | No |

# | Penetration (Y5) | Beta(α, β) scaled to \[min, max] — by revenue tier | No |

# | Regulatory delay | Bernoulli(p) + Uniform(min\_months, max\_months) — by Africa experience | No |

# 

# \### Correlation Implementation

# 

# Draw 3 independent standard normals Z₁, Z₂, Z₃. Apply Cholesky L. Transform each to target marginal via inverse CDF.

# 

# Impact: Correlation widens P5–P10 by \~25–40% and P90–P95 by \~15–25% vs. independence.

# 

# \### Sigmoid Penetration Ramp

# 

# ```

# penetration(year) = final\_pen × 1/(1 + exp(-k × (effective\_year - c)))

# ```

# 

# k=2.5 (steepness), c=2.5 (center). effective\_year = max(0, year − delay\_years).

# 

# Schedule: Y1≈10%, Y2≈38%, Y3≈73%, Y4≈92%, Y5≈98% of final.

# 

# \*\*Sensitivity:\*\* k from 1.5–3.5 → ±30–40% on Y3. c from 2.0–3.0 → ±20–25% on Y3. Disclose as modeling choices.

# 

# \### FX Impact

# 

# fx\_impact = 10.8 / drawn\_rate. σ\_log ÷ √year for mean reversion.

# 

# \*\*Mandatory disclosure:\*\* "FX volatility assumed to decrease over horizon. Ghana 2020–2025 data shows episodic volatility (stable periods + rapid depreciation events) rather than smooth mean reversion. May understate late-horizon FX shocks."

#

#

# |---|---|---|---|

# | <$1M | 2 | 12 | \~0.5%–6% |

# | $1M–$10M | 2 | 8 | \~1.5%–12% |

# | $10M–$50M | 2.5 | 6 | \~2%–18% |

# | $50M–$250M | 3 | 5 | \~3%–22% |

# | $250M+ | 3 | 4 | \~4%–28% |
